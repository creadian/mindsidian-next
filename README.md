# Mindsidian Next (v2 prototype)

This is a from-scratch rewrite of the Mindsidian mind-map plugin for Obsidian, built so that data safety (no markdown corruption, no data loss) outranks every feature. The authoritative build spec is `docs/05-DESIGN.md`; the markdown format it must honor byte-for-byte is `docs/03-format-contract.md`. The core (parser, serializer, tree commands) is pure TypeScript with no Obsidian dependencies and is fully covered by golden round-trip tests before any UI exists. Every save passes a refuse-to-corrupt guard: if the serialized result doesn't re-parse into the same tree, the plugin writes the last known good text instead of touching your note.

To try it safely side-by-side with the working v0.5.47 plugin: run `npm install` then `npm run build`, and copy `main.js`, `manifest.json`, and `styles.css` into `Claude_testing/.obsidian/plugins/mindsidian-next/` (the test vault — never the real Brain Forest vault). Because the plugin id is `mindsidian-next`, it installs alongside the original `mindsidian` plugin without touching it; enable it in Settings → Community plugins. Your existing maps stay owned by v1 until you explicitly open one with the v2 view, and the prototype's shadow-validation command lets you check that v2 round-trips every mindmap file in the vault without writing anything.
